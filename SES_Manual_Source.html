1. The SES Philosophy

Simulink Easy Script (SES) was born from a simple observation: Simulink is powerful, but its native scripting API is slow and verbose.

SES provides a declarative, Pythonic layer that abstracts away the complexity of block creation, parameter setting, and wiring. By decoupling the “Build” phase from the “Render” phase, SES achieves speeds 20x faster than standard MATLAB scripts.

2. Core Engine API Reference

start(modelName)

The entry point for any SES session. - Background Loading: Uses load_system to build models in memory. - Environment Scrubbing: Clears shadowing variables and old .slx files.

connect(src, dst)

Intelligent port-to-port wiring. - Supports BlockName.PortIndex notation. - Resolves hierarchical paths in complex subsystems automatically.

setparam(block, parameter, value)

The standard configuration interface. - Value Casting: All values should be passed as strings (e.g., '10', 'on', '[1 2 1]').

3. The Block Library: A Detailed Reference

3.1 Continuous Package

Function Simulink Path Typical Use Case
integrator(name) simulink/Continuous/Integrator State-space models, physical systems.
pid(name) simulink/Continuous/PID Controller Closed-loop feedback control.
transferfcn(name) simulink/Continuous/Transfer Fcn Frequency domain modeling.

3.2 Discrete Package

Function Simulink Path Typical Use Case
delay(name) simulink/Discrete/Unit Delay Digital signal processing.
filter(name) simulink/Discrete/Filter IIR/FIR filter implementation.

3.3 Logic and Bit Operations

  • logical(name): Single block for AND, OR, XOR, NAND, NOR, NOT.
  • compare(name): Single block for ==, ~=, <, >, <=, >=.

4. The Cookbook: Professional Design Patterns 🍳

4.1 Second-Order Transfer Function

To model a system with $G(s) = \frac{1}{s^2 + 2s + 1}$:

from ses import setup transferfcn step scope
setup()
start("Control_Demo")

sys = transferfcn("G");
setparam(sys, 'Numerator', '[1]');
setparam(sys, 'Denominator', '[1 2 1]');

connect("Step.1", "G.1")
run()

4.2 Feedback Control System

from ses import setup sine sum pid integrator scope
setup()
start("Feedback_Loop")

sine("Setpoint")
sum("Comparator")
setparam("Comparator", 'Inputs', '+-')

pid("Controller")
setparam("Controller", 'P', '10', 'D', '0.1')

integrator("Plant")
scope("Monitor")

% High-Level Wiring
connect("Setpoint.1", "Comparator.1")
connect("Comparator.1", "Controller.1")
connect("Controller.1", "Plant.1")
connect("Plant.1", "Monitor.1")
connect("Plant.1", "Comparator.2") % Feedback

arrange()
run()

5. Advanced: Parameter Deep-Dive

SES allows you to tune every hidden property of a Simulink block. 1. Open the block’s “Block Parameters” window in the GUI once. 2. The field label (e.g., “Proportional (P)”) often differs from the internal name (e.g., P). 3. Use get_param(gcb, 'ObjectParameters') to find the exact key for setparam.

6. Performance & Scale

SES scales linearly. Even at 500 blocks, the build time remains under 30 seconds, a task that would take over 10 minutes using standard add_block calls with the GUI open.
